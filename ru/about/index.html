О SwimAnalytics

Научное отслеживание результатов в плавании, созданное пловцами для пловцов

Наша миссия

SwimAnalytics предоставляет отслеживание результатов профессионального уровня каждому пловцу. Мы считаем, что продвинутые метрики, такие как критическая скорость плавания (CSS), оценка тренировочного стресса (TSS) и графики управления производительностью, не должны быть заблокированы за дорогими платформами или требовать сложного тренерского программного обеспечения.

Научная основа

SwimAnalytics построен на десятилетиях рецензируемых исследований в спортивной науке:

Критическая скорость плавания (CSS)

Основана на исследованиях Wakayoshi et al. (1992-1993) в Университете Осаки. CSS представляет максимальную теоретическую скорость плавания, поддерживаемую без истощения, соответствующую лактатному порогу.

Ключевое исследование: Wakayoshi K, et al. "Определение и валидность критической скорости как индекса производительности плавания." European Journal of Applied Physiology, 1992.

Оценка тренировочного стресса (TSS)

Адаптирована из методологии TSS для велоспорта доктора Эндрю Когган для плавания. Количественно определяет тренировочную нагрузку, сочетая интенсивность (относительно CSS) и продолжительность.

Ключевое исследование: Coggan AR, Allen H. "Тренировки и гонки с измерителем мощности." VeloPress, 2010. Адаптировано для плавания SwimAnalytics с использованием CSS в качестве порога.

График управления производительностью (PMC)

Метрики хронической тренировочной нагрузки (CTL), острой тренировочной нагрузки (ATL) и баланса тренировочного стресса (TSB). Отслеживает физическую форму, усталость и форму с течением времени.

Реализация: Экспоненциально взвешенное скользящее среднее 42 дней для CTL, 7 дней для ATL. TSB = CTL - ATL.

SWOLF и метрики гребка

Метрики эффективности плавания, сочетающие время и количество гребков. Используются элитными пловцами и тренерами по всему миру для отслеживания технических улучшений.

Стандартные метрики: SWOLF = Время + Гребки. Более низкие оценки указывают на лучшую эффективность. Дополняется расстоянием за гребок (DPS) и частотой гребка (SR).

Разработка и обновления

SwimAnalytics активно разрабатывается с регулярными обновлениями на основе отзывов пользователей и последних исследований в спортивной науке. Приложение создано с использованием:

  • Swift и SwiftUI - современная нативная разработка для iOS
  • Интеграция с HealthKit - безупречная синхронизация с Apple Health
  • Core Data - эффективное локальное хранение данных
  • Swift Charts - красивые и интерактивные визуализации данных
  • Без сторонней аналитики - ваши данные об использовании остаются конфиденциальными
