Începe cu SwimAnalytics

Ghidul tău complet pentru urmărirea performanței în înot, testare CSS și analiza sarcinii de antrenament

Bine ai Venit în Înotul Bazat pe Date

SwimAnalytics transformă antrenamentele tale de înot în perspective acționabile folosind metricile Critical Swim Speed (CSS), Training Stress Score (sTSS) și Performance Management Chart (PMC). Acest ghid te va duce de la configurarea inițială până la analiza avansată a sarcinii de antrenament în 4 pași simpli.

Start Rapid (5 Minute)

1

Descarcă și Instalează

Descarcă SwimAnalytics din App Store și acordă permisiunea de acces la Apple Health. Aplicația sincronizează automat antrenamentele de înot—nu este necesară înregistrarea manuală.

Descarcă Aplicația →
2

Efectuează Testul CSS

Completează un test cronometrat de 400m și 200m pentru a stabili Viteza Ta Critică de Înot. Aceasta este baza tuturor metricilor—fără CSS, nu se poate calcula sTSS sau zonele de antrenament.

Protocol Test CSS ↓
3

Introdu Rezultatele CSS

Introdu timpii tăi de 400m și 200m în aplicație. SwimAnalytics calculează CSS, zonele de tempo și personalizează toate metricile pe fiziologia ta. Actualizează la fiecare 6-8 săptămâni pe măsură ce forma fizică se îmbunătățește.

4

Începe să Înregistrezi Antrenamente

Înoată cu Apple Watch și aplicația Sănătate. SwimAnalytics importă automat antrenamentele, calculează sTSS, actualizează CTL/ATL/TSB și urmărește progresul. Nu este necesară introducerea manuală a datelor.

Protocol Complet Test CSS

📋 Ce Ai Nevoie

  • Acces la piscină: Bazin de 25m sau 50m (25yd acceptabil)
  • Cronometrare: Cronometru, ceas de piscină sau Apple Watch
  • Timp de încălzire: 15-20 minute înainte de test
  • Recuperare: 5-10 minute între eforturi
  • Efort: Tempo maxim sustenabil (nu sprint total)

⏱️ Condiții pentru Ziua Testului

  • Odihnit: Fără antrenament intens 24-48 ore înainte
  • Hidratat: Bine hidratat, nutriție normală
  • Temperatura piscinei: 26-28°C ideal (evită prea rece/cald)
  • Ora zilei: Când înoți de obicei cel mai bine
  • Echipament: Același ca la antrenamente (ochelari, cască, costum)

Testul CSS Pas cu Pas

Încălzire

15-20 minute

400-800m înot ușor, exerciții tehnice și progresive. Include 2-3×50 la tempo crescând (60%, 75%, 85% efort). Odihnește-te 2-3 minute înainte de test.

Test 1

400m Efort Maxim

Start de pe perete (fără săritură). Înoată 400m la cel mai rapid tempo pe care îl poți susține pentru întreaga distanță. Acesta NU este un sprint—gestionează-ți efortul. Înregistrează timpul în format mm:ss (ex: 6:08).

Sfat pentru tempo: Încearcă să menții fracții uniforme de 100m. Ultimii 200m ar trebui să fie ≤ primii 200m (negativ ideal).
Recuperare

5-10 minute

FAZĂ CRITICĂ: Înot ușor sau odihnă completă. Așteaptă până când ritmul cardiac scade sub 120 bpm și respirația se recuperează complet. Recuperare insuficientă = CSS imprecis.

Test 2

200m Efort Maxim

Start de pe perete (fără săritură). Efort maxim sustenabil pentru 200m. Acesta ar trebui să se simtă mai greu pe 100m decât 400m. Înregistrează timpul în format mm:ss (ex: 2:30).

Verificare validitate: Tempoul 200m/100m ar trebui să fie 3-6 secunde mai rapid decât tempoul 400m/100m. Dacă nu, recuperarea a fost insuficientă sau tempoul inadecvat.
Răcire

10-15 minute

300-500m înot ușor, stretching. Înregistrează-ți timpii imediat—nu te baza pe memorie.

⚠️ Erori Comune în Testul CSS

  • Start prea rapid în 400m: Rezultă în colaps, CSS imprecis. Folosește tempo uniform.
  • Recuperare insuficientă între eforturi: Oboseala încetinește 200m, făcând CSS artificial rapid → zone de supraantrenament.
  • Folosirea starturilor cu săritură: Adaugă avantaj de 0,5-1,5s, distorsionând calculele. Startează întotdeauna de pe perete.
  • Testare când ești obosit: Sarcină grea de antrenament 24-48h înainte = rezultate depresive. Testează când ești odihnit.
  • Nu înregistrezi imediat: Memoria nu este de încredere. Notează timpii înainte de răcire.

Introducerea Rezultatelor CSS în SwimAnalytics

Pasul 1: Deschide Configurarea CSS

În aplicația SwimAnalytics, navighează la Setări → Critical Swim Speed. Atinge "Efectuează Test CSS" sau "Actualizează CSS".

Pasul 2: Introdu Timpii

Introdu timpul tău de 400m (ex: 6:08) și timpul de 200m (ex: 2:30). Folosește formatul exact arătat. Atinge "Calculează".

Pasul 3: Revizuiește Rezultatele

Aplicația afișează:

  • Viteza CSS: 0,917 m/s
  • Tempo CSS: 1:49/100m
  • Zone de antrenament: 7 zone personalizate (Zona 1-7)
  • Baza sTSS: Acum activată pentru toate antrenamentele

Pasul 4: Salvează și Sincronizează

Atinge "Salvează CSS". Aplicația imediat:

  • Recalculează zonele de antrenament
  • Actualizează retroactiv sTSS pentru ultimele 90 zile
  • Ajustează calculele CTL/ATL/TSB
  • Activează analiza antrenamentelor pe zone

💡 Sfat Pro: Teste CSS Istorice

Dacă știi deja CSS-ul tău din teste anterioare, poți introduce acei timpi direct. Cu toate acestea, pentru rezultate mai precise, efectuează un test nou la fiecare 6-8 săptămâni. CSS-ul tău ar trebui să se îmbunătățească (să devină mai rapid) pe măsură ce antrenamentul progresează.

Înțelegerea Metricilor Tale

Critical Swim Speed (CSS)

Ce este: Tempoul tău de prag aerobic—viteza cea mai rapidă pe care o poți susține pentru ~30 minute fără epuizare.

Ce înseamnă: CSS = 1:49/100m înseamnă că poți menține tempo de 1:49 în eforturi susținute de prag.

Cum să-l folosești: Baza pentru toate zonele de antrenament și calculul sTSS. Actualizează la fiecare 6-8 săptămâni.

Află despre CSS →

Zone de Antrenament

Ce sunt: 7 intervale de intensitate bazate pe CSS-ul tău, de la recuperare (Zona 1) la sprint (Zona 7).

Ce înseamnă: Fiecare zonă vizează adaptări fiziologice specifice (bază aerobică, prag, VO₂max).

Cum să le folosești: Urmează prescripțiile de zonă pentru antrenament structurat. Aplicația arată timpul în zonă pentru fiecare antrenament.

Zone de Antrenament →

Swimming Training Stress Score (sTSS)

Ce este: Stresul de antrenament cuantificat combinând intensitatea și durata. 1 oră la tempo CSS = 100 sTSS.

Ce înseamnă: sTSS 50 = recuperare ușoară, sTSS 100 = moderat, sTSS 200+ = sesiune foarte grea.

Cum să-l folosești: Urmărește sTSS zilnic/săptămânal pentru a gestiona sarcina de antrenament. Vizează creștere de 5-10 sTSS pe săptămână ca maxim.

Ghid sTSS →

CTL / ATL / TSB

Ce sunt:

  • CTL: Chronic Training Load (formă fizică) - medie de 42 zile a sTSS
  • ATL: Acute Training Load (oboseală) - medie de 7 zile a sTSS
  • TSB: Training Stress Balance (formă) = CTL - ATL

Cum să le folosești: TSB pozitiv = proaspăt/în coborâre, TSB negativ = obosit. Concurează când TSB = +5 la +25.

📊 Obiectivele Primei Săptămâni

După introducerea CSS și completarea a 3-5 antrenamente:

  • Verifică valorile sTSS: Confirmă că se potrivesc cu percepția efortului (ușor ~50, moderat ~100, greu ~150+)
  • Revizuiește distribuția pe zone: Petreci 60-70% în Zona 2 (bază aerobică)?
  • Stabilește CTL de bază: Media sTSS din prima săptămână devine linia de bază inițială a formei fizice
  • Identifică tiparele: Ce antrenamente generează cel mai mult sTSS? Te recuperezi adecvat?

Parcursul Tipic al Utilizatorului (Primele 8 Săptămâni)

Săptămâna 1-2: Stabilirea Liniei de Bază

  • Efectuează testul CSS și introdu rezultatele
  • Completează 3-5 antrenamente normale
  • Observă valorile sTSS și distribuția pe zone
  • Stabilește CTL inițial (nivel de formă fizică)
  • Obiectiv: Înțelege metricile, nicio schimbare încă

Săptămâna 3-4: Aplicarea Zonelor

  • Folosește zonele CSS în planificarea antrenamentelor
  • Înoată intenționat în Zona 2 pentru serii aerobice
  • Urmărește totalurile sTSS săptămânale (caută consistență)
  • Monitorizează TSB (ar trebui să fie ușor negativ = antrenament)
  • Obiectiv: Antrenează-te pe zone, nu pe senzație

Săptămâna 5-6: Supraîncărcare Progresivă

  • Crește sTSS săptămânal cu 5-10% de la linia de bază
  • Adaugă 1 sesiune de prag (Zona 4) pe săptămână
  • CTL ar trebui să crească treptat (formă fizică în îmbunătățire)
  • ATL poate avea vârfuri în săptămâni grele (normal)
  • Obiectiv: Progresie controlată a formei fizice

Săptămâna 7-8: Reevaluare și Ajustare

  • Efectuează al doilea test CSS (ar trebui să fie mai rapid)
  • Actualizează zonele în aplicație (tempoul se îmbunătățește)
  • Compară CTL Săptămâna 1 vs Săptămâna 8 (ar trebui +10-20)
  • Revizuiește progresul: Timpii scad? Se simte mai ușor?
  • Obiectiv: Validează eficacitatea antrenamentului

✅ Indicatori de Succes

După 8 săptămâni de antrenament structurat cu SwimAnalytics, ar trebui să vezi:

  • Îmbunătățire CSS: 1-3% mai rapid în tempo CSS (ex: 1:49 → 1:47)
  • Creștere CTL: +15-25 puncte (ex: 30 → 50 CTL)
  • sTSS consistent: Totaluri săptămânale în 10-15% variație
  • Ritm mai bun: Fracții mai uniforme, calibrare mai bună a efortului
  • Recuperare îmbunătățită: TSB ciclează previzibil (-10 la +5)

Depanare și Întrebări Frecvente

sTSS-ul meu pare prea mare/mic pentru efortul antrenamentului

Cauză: CSS-ul este învechit sau imprecis.

Soluție: Repetă testul CSS. Dacă ai testat când erai obosit sau ai gestionat prost tempoul, CSS-ul va fi greșit. Un test CSS corect este critic pentru toate metricile ulterioare.

Aplicația afișează "Niciun CSS configurat"

Cauză: Test CSS necompletat sau nesalvat.

Soluție: Navighează la Setări → Critical Swim Speed → Efectuează Test. Introdu ambii timpi de 400m și 200m, apoi atinge Salvează.

Antrenamentele nu se sincronizează de pe Apple Watch

Cauză: Permisiuni aplicație Sănătate neacordate sau antrenament necategorizat ca "Înot".

Soluție: Verifică Setări → Confidențialitate → Sănătate → SwimAnalytics → Permite Citire Antrenamente. Asigură-te că tipul de antrenament pe Apple Watch este "Înot în Piscină" sau "Înot în Ape Deschise".

CTL nu crește în ciuda antrenamentului consistent

Cauză: Totaluri sTSS prea mici sau frecvență inconsistentă.

Soluție: CTL este o medie ponderată exponențial de 42 zile. Crește lent. Crește sTSS săptămânal cu 5-10%, și menține 4+ antrenamente/săptămână pentru creștere consistentă a CTL.

Cât de des ar trebui să retestez CSS?

Recomandare: La fiecare 6-8 săptămâni în fazele de bază/construcție. Retestează după boală, accidentare, pauză lungă, sau când zonele se simt constant prea ușoare/grele.

Pot folosi SwimAnalytics pentru alte stiluri?

Da, cu limitări: CSS-ul se testează de obicei în stil liber. Pentru antrenamente IM/spate/bras, sTSS este calculat pe baza CSS de stil liber. Consideră efectuarea testelor CSS specifice stilului pentru precizie mai mare.
