Comenzar dengan SwimAnalytics

Anda guía completa untuk pelacakan performa di renang, pruebas CSS dan analitik beban latihan

Bienvenido Natación Basada di Datos

SwimAnalytics transforma Anda latihans renang di informasi práctica utilizando metrik Critical Swim Speed (CSS), Skor Stres Latihan (sTSS) dan Grafik Manajemen Performa (PMC). Esta guía te llevará dari configuración inicial hingga analitik avanzado beban latihan di 4 pasos sencillos.

Inicio Rápido (5 Minutos)

1

Unduh e Instalar

Descarga SwimAnalytics dari App Store dan concede permiso untuk acceder Apple Health. La app sincroniza latihans renang automáticamente: tidak se requiere registro manual.

Unduh App →
2

Realizar Prueba CSS

Completa prueba contrarreloj 400m dan 200m untuk establecer Anda Critical Swim Speed. Esta es base semua metrik: tanpa CSS, tidak se pueden calcular sTSS ni zona latihan.

Protocolo Prueba CSS ↓
3

Introducir Resultados CSS

Introduce Anda tiempos 400m dan 200m di app. SwimAnalytics calcula CSS, zona ritmo dan personaliza semua metrik según Anda fisiología. Actualiza setiap 6-8 semanas medida yang mejora Anda estado forma.

4

Comenzar Registrar Latihans

Nada dengan Apple Watch dan app Health. SwimAnalytics importa automáticamente latihans, calcula sTSS, actualiza CTL/ATL/TSB dan realiza pelacakan progreso. No se necesita entrada manual data.

Protocolo Completo Prueba CSS

📋 Lo Que Necesitas

  • Acceso piscina: Piscina 25m atau 50m (25yd aceptable)
  • Cronometraje: Cronómetro, reloj pared atau Apple Watch
  • Waktu calentamiento: 15-20 minutos antes prueba
  • Recuperación: 5-10 minutos entre pruebas
  • Esfuerzo: Ritmo máximo sostenible (no sprint total)

⏱️ Condiciones Día Prueba

  • Descansado: Sin latihan duro 24-48 horas antes
  • Hidratado: Bien hidratado, alimentación normal
  • Temperatura piscina: 26-28°C ideal (evitar muy fría/caliente)
  • Hora día: Cuando normalmente entrenas mejor
  • Equipamiento: Igual yang di latihans (gafas, gorro, bañador)

Prueba CSS Paso Paso

Calentamiento

15-20 minutos

400-800m renang suave, ejercicios técnicos dan progresivos. Incluye 2-3×50 ritmo creciente (60%, 75%, 85% esfuerzo). Descansa 2-3 minutos antes prueba.

Prueba 1

400m Esfuerzo Máximo

Salida dari pared (sin zambullida). Nada 400m al ritmo lebih dari rápido yang puedas sostener durante toda distancia. Esto NO adalah sprint: dosifica Anda esfuerzo. Registra tiempo di formato mm:ss (ej.: 6:08).

Consejo ritmo: Intenta mantener parciales 100m uniforbulan. Los segundos 200m deberían ser ≤ primeros 200m (negativo ideal).
Recuperación

5-10 minutos

FASE CRÍTICA: Renang suave atau descanso completo. Espera hingga yang frecuencia cardíaca baje 120 ppm dan respiración se recupere completamente. Recuperación insuficiente = CSS inexacto.

Prueba 2

200m Esfuerzo Máximo

Salida dari pared (sin zambullida). Esfuerzo sostenible máximo durante 200m. Esto debería sentirse lebih dari duro oleh setiap 100m yang 400m. Registra tiempo di formato mm:ss (ej.: 2:30).

Lihatificación validez: El ritmo 200m/100m debería ser 3-6 segundos lebih dari rápido yang ritmo 400m/100m. Si tidak es así, recuperación fue insuficiente atau ritmo inadecuado.
Vuelta calma

10-15 minutos

300-500m renang suave, estiramientos. Registra Anda tiempos inrata-ratatamente: tidak confíes di memoria.

⚠️ Errores Comunes di Prueba CSS

  • Salir demasiado rápido di 400m: Resulta di desfondamiento, CSS inexacto. Usa ritmo uniforme.
  • Recuperación insuficiente entre pruebas: La fatiga ralentiza 200m, haciendo yang CSS sea artificialmente rápido → zona tentanglatihan.
  • Usar salidas dengan zambullida: Añade ventaja 0.5-1.5s, sesgando cálculos. Siempre sal dari pared.
  • Probar ketika adalahs fatigado: Carga latihan pesada 24-48h antes = resultados deprimidos. Prueba ketika estés fresco.
  • No registrar inrata-ratatamente: La memoria tidak es fiable. Anota tiempos antes vuelta calma.

Introducir Resultados CSS di SwimAnalytics

Paso 1: Abrir Configuración CSS

En app SwimAnalytics, ve Pengaturan → Critical Swim Speed. Toca "Realizar Prueba CSS" atau "Actualizar CSS".

Paso 2: Introducir Waktus

Introduce Anda tiempo 400m (ej.: 6:08) dan tiempo 200m (ej.: 2:30). Usa formato exacto mostrado. Toca "Calcular".

Paso 3: Revisar Resultados

La app muestra:

  • Kecepatan CSS: 0.917 m/s
  • Ritmo CSS: 1:49/100m
  • Zona latihan: 7 zona personalizadas (Zona 1-7)
  • Línea base sTSS: Ahora habilitada untuk semua latihans

Paso 4: Guardar dan Sincronizar

Toca "Guardar CSS". La app inrata-ratatamente:

  • Recalcula zona latihan
  • Actualiza retrosecara aktif sTSS últimos 90 días
  • Ajusta cálculos CTL/ATL/TSB
  • Habilita analitik latihans oleh zona

💡 Consejo Pro: Pruebas Históricas CSS

Si ya conoces Anda CSS pruebas anteriores, puedes introducir esos tiempos directamente. Sin embargo, untuk obtener resultados lebih dari precisos, realiza prueba nueva setiap 6-8 semanas. Anda CSS debería mejorar (volverse lebih dari rápido) medida yang progritu latihan.

Comprender Tus Metrik

Critical Swim Speed (CSS)

Qué es: Anda ritmo umbral aeróbico: kecepatan lebih dari rápida yang puedes sostener durante ~30 minutos tanpa agotamiento.

Qué significa: CSS = 1:49/100m significa yang puedes mantener ritmo 1:49 di esfuerzos sostenidos umbral.

Cómo usarlo: Base untuk semua zona latihan dan cálculo sTSS. Actualiza setiap 6-8 semanas.

Aprende CSS →

Zona Latihan

Qué son: 7 rangos intensidad berdasarkans di Anda CSS, dari recuperación (Zona 1) hingga sprint (Zona 7).

Qué significan: Cada zona apunta adaptaciones fisiológicas específicas (base aeróbica, umbral, VO₂máx).

Cómo usarlas: Sigue prescripciones zona untuk latihan estructurado. La app muestra tiempo di zona untuk setiap latihan.

Zona Latihan →

Skor Stres Latihan di Natación (sTSS)

Qué es: Stres cuantificado latihan combinando intensidad dan duración. 1 hora ritmo CSS = 100 sTSS.

Qué significa: sTSS 50 = recuperación fácil, sTSS 100 = moderado, sTSS 200+ = sesión muy dura.

Cómo usarlo: Rastrea sTSS diario/semanal untuk gestionar beban latihan. Apunta aumento 5-10 sTSS oleh semana sebagai máximo.

Guía sTSS →

CTL / ATL / TSB

Qué son:

  • CTL: Chronic Training Load (fitness) - promedio 42 días sTSS
  • ATL: Beban Latihan Aguda (fatiga) - promedio 7 días sTSS
  • TSB: Keseimbangan Stres Latihan (forma) = CTL - ATL

Cómo usarlos: TSB positivo = fresco/en puini punto, TSB negativo = fatigado. Compite ketika TSB = +5 +25.

📊 Objetivos Anda Primera Semana

Setelah introducir CSS dan completar 3-5 latihans:

  • Lihatificar valores sTSS: Confirma yang coincidan dengan percepción esfuerzo (fácil ~50, moderado ~100, duro ~150+)
  • Revisar distribución oleh zona: ¿Estás pasando 60-70% di Zona 2 (base aeróbica)?
  • Establecer CTL base: El promedio sTSS Anda primera semana se convierte di línea base fitness inicial
  • Identificar patrones: ¿Qué latihans generan mayor sTSS? ¿Te adalahs recuperando adecuadamente?

Recorrido Típico Usuario (Primeras 8 Semanas)

Semana 1-2: Establecer Línea Base

  • Realizar prueba CSS e introducir resultados
  • Completar 3-5 latihans normales
  • Observar valores sTSS dan distribución oleh zona
  • Establecer CTL inicial (nivel fitness)
  • Objetivo: Comprender metrik, tanpa cambios aún

Semana 3-4: Aplicar Zona

  • Usar zona CSS di planificación latihans
  • Nadar intencionadamente di Zona 2 untuk series aeróbicas
  • Rastrear totales sTSS semanales (buscar consistencia)
  • Monitorear TSB (debería ser ligeramente negativo = entrenando)
  • Objetivo: Entrenar oleh zona, tidak oleh sensación

Semana 5-6: Sobrecarga Progresiva

  • Aumentar sTSS semanal 5-10% dari línea base
  • Añadir 1 sesión umbral (Zona 4) oleh semana
  • CTL debería subir gradualmente (fitness mejorando)
  • ATL puede pico di semanas duras (normal)
  • Objetivo: Progresión controlada fitness

Semana 7-8: Re-evaluar dan Ajustar

  • Realizar segunda prueba CSS (debería ser lebih dari rápida)
  • Actualizar zona di app (el ritmo mejora)
  • Comparar CTL Semana 1 vs Semana 8 (debería ser +10-20)
  • Revisar progreso: ¿Están bajando tiempos? ¿Se siente lebih dari fácil?
  • Objetivo: Validar efectividad latihan

✅ Indicadores Éxito

Setelah 8 semanas latihan estructurado dengan SwimAnalytics, deberías ver:

  • Mejora di CSS: 1-3% lebih dari rápido di ritmo CSS (ej.: 1:49 → 1:47)
  • Aumento CTL: +15-25 puntos (ej.: 30 → 50 CTL)
  • sTSS consistente: Totales semanales dentro 10-15% varianza
  • Mejor dosificación: Parciales lebih dari uniforbulan, mejor calibración esfuerzo
  • Recuperación mejorada: TSB cicla predeciblemente (-10 +5)

Solución Problemas dan Pertanyaan Umum

Mi sTSS parece demasiado alto/bajo untuk esfuerzo latihan

Causa: El CSS adalah desactualizado atau es inexacto.

Solución: Vuelve probar CSS. Si probaste ketika estabas fatigado atau dosificaste mal, CSS será incorrecto. Una prueba CSS adecuada es kritis untuk semua metrik posteriores.

La app muestra "Tidak ada CSS configurado"

Causa: Prueba CSS tidak completada atau tidak guardada.

Solución: Ve Pengaturan → Critical Swim Speed → Realizar Prueba. Introduce ambos tiempos 400m dan 200m, luego toca Guardar.

Los latihans tidak se sincronizan dari Apple Watch

Causa: Permisos app Health tidak concedidos atau latihan tidak categorizado sebagai "Natación".

Solución: Lihatifica Pengaturan → Privacidad → Salud → SwimAnalytics → Permitir Lectura Latihans. Asegúrate yang tipo latihan di Apple Watch sea "Natación di Piscina" atau "Natación di Aguas Abiertas".

El CTL tidak aumenta pesar latihan consistente

Causa: Totales sTSS demasiado bajos atau frecuencia inconsistente.

Solución: CTL adalah promedio ponderado exponencialmente 42 días. Sube lentamente. Aumenta sTSS semanal 5-10%, dan mantén 4+ latihans/semana untuk crecimiento consistente CTL.

¿Con qué frecuencia debería volver probar CSS?

Recomendación: Cada 6-8 semanas durante fases base/construcción. Vuelve probar setelah enfermedad, lesión, pausa larga, atau ketika zona se sientan consistentemente demasiado fáciles/difíciles.

¿Puedo usar SwimAnalytics untuk otros estilos?

Sí, dengan limitaciones: El CSS típicamente se prueba di estilo libre. Para latihans IM/espalda/braza, sTSS se calcula basándose di CSS estilo libre. Considera realizar pruebas CSS específicas oleh estilo untuk mayor ketepatan.
