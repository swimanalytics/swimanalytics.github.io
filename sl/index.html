SwimAnalytics - Analisi Scientifica del Nuoto

Metriche basate su CSS, sTSS e Performance Management Chart per nuotatori seri

Nuoto Basato sui Dati per Prestazioni Serie

SwimAnalytics porta l'analisi di allenamento di livello professionale al nuoto, utilizzando Critical Swim Speed (CSS), swimming Training Stress Score (sTSS) e Performance Management Chart (PMC) per trasformare i tuoi allenamenti in progressi misurabili.

📊 Basato su Metriche Scientifiche

Ogni calcolo è fondato su ricerca scientifica peer-reviewed, non su stime approssimative. CSS determina le tue zone di allenamento personalizzate. sTSS quantifica lo stress di allenamento. PMC traccia fitness, fatica e forma.

🎯 Allenamento Personalizzato

Le tabelle di andatura generiche falliscono. Le zone CSS si adattano alla TUA fisiologia. L'andatura di soglia di un nuotatore è l'andatura di recupero di un altro. Allenati alla tua intensità ideale, sempre.

📈 Traccia Progressi a Lungo Termine

CTL (fitness cronico), ATL (fatica acuta) e TSB (bilanciamento dello stress) rivelano quando stai costruendo forma fisica, quando sei sovrallenato e quando sei pronto per gareggiare al massimo.

⚡ Integrazione Automatica

Nuota con Apple Watch. SwimAnalytics sincronizza automaticamente via Apple Health, calcola sTSS, aggiorna PMC e traccia il tempo nelle zone. Zero inserimento manuale dei dati.

Domande Frequenti

Cos'è il CSS e perché è importante?

Critical Swim Speed (CSS) è la tua velocità di soglia aerobica: l'andatura più veloce che puoi sostenere per ~30 minuti senza accumulo di lattato. È fondamentale perché:

  • Determina tutte le zone di allenamento personalizzate
  • Serve come base per il calcolo di sTSS
  • Predice la prestazione nelle gare di media distanza
  • Rappresenta il confine tra esercizio aerobico e anaerobico

A differenza delle tabelle di andatura generiche, il CSS si adatta alla TUA fisiologia. Il CSS di un nuotatore potrebbe essere 1:20/100m, mentre quello di un altro è 2:00/100m, ma entrambi possono allenarsi efficacemente usando le zone basate sul CSS.

Come si calcola il swimming Training Stress Score (sTSS)?

sTSS quantifica lo stress di allenamento combinando intensità e durata:

sTSS = (IF³) × Durata (ore) × 100

Dove il Fattore di Intensità (IF) = Andatura CSS / Andatura Media dell'Allenamento. Il fattore cubico (IF³) tiene conto della resistenza esponenziale dell'acqua: nuotare il 10% più veloce richiede ~33% in più di potenza.

Esempio: 60 min all'andatura CSS (IF=1.0) = 100 sTSS. 60 min al 110% del CSS (più lento, IF=0.91) = 75 sTSS.

Cosa significano CTL, ATL e TSB?

Queste tre metriche del Performance Management Chart (PMC) raccontano la storia completa del tuo allenamento:

  • CTL (Chronic Training Load): Il tuo livello di fitness (media esponenziale ponderata di 42 giorni di sTSS). CTL più alto = più in forma.
  • ATL (Acute Training Load): La tua fatica (media esponenziale ponderata di 7 giorni di sTSS). ATL alto = affaticato dal recente allenamento.
  • TSB (Training Stress Balance): La tua forma (CTL - ATL). TSB positivo = fresco, TSB negativo = carico di allenamento.

Applicazione Pratica: Gareggia quando TSB = +15 a +25 (fresco ma in forma). Allena quando TSB = -10 a -25 (costruzione fitness).

Ogni quanto dovrei testare nuovamente il CSS?

Ogni 6-8 settimane durante le fasi di base e costruzione. Il tuo CSS dovrebbe migliorare (diventare più veloce) man mano che il fitness migliora. Testa nuovamente dopo:

  • Malattia o infortunio (il CSS potrebbe regredire)
  • Pause lunghe dall'allenamento (detraining)
  • Quando le zone sembrano costantemente troppo facili/difficili
  • Prima di un periodo di gare importanti (assicura zone accurate)

Il CSS non è una metrica statica: si adatta con il tuo fitness. Aggiornare regolarmente garantisce che tutte le zone e i calcoli di sTSS rimangano accurati.

SwimAnalytics funziona con nuoto in acque libere o solo in piscina?

SwimAnalytics funziona con entrambi gli allenamenti in piscina e in acque libere registrati da Apple Watch:

  • Nuoto in Piscina: Pieno supporto. Apple Watch traccia automaticamente vasche, andatura, SWOLF, bracciate. SwimAnalytics calcola sTSS, tempo nelle zone e tutte le metriche.
  • Nuoto in Acque Libere: Supportato. Apple Watch usa GPS per tracciare distanza/andatura. I calcoli di sTSS e zone funzionano normalmente. Dati SWOLF/bracciate limitati (nessun rilevamento vasche).

Nota sul test CSS: Esegui il test in piscina per risultati accurati. L'acque libere hanno correnti/onde che influenzano l'andatura.

Posso usare SwimAnalytics per stili diversi dallo stile libero?

Sì, con limitazioni: Il CSS viene tipicamente testato in stile libero (il più efficiente dal punto di vista metabolico). Per altri stili:

  • Allenamenti di Medley/Dorso/Rana: sTSS viene calcolato basandosi sul CSS di stile libero. Regola mentalmente: dorso ~105% del CSS, rana ~110-120% del CSS.
  • Test CSS specifici per stile: Puoi eseguire test CSS separati per dorso/rana (400m + 200m nello stesso stile). Inserisci quello nella app per zone specifiche per stile.
  • Delfino: Non raccomandato per test CSS (troppo anaerobico). Usa zone di stile libero come riferimento.

Per la maggior parte degli utenti, le zone CSS di stile libero si generalizzano abbastanza bene per l'allenamento in stili misti.
