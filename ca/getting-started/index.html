Començar amb SwimAnalytics

La teva guia completa per fer seguiment del rendiment en natació, proves CSS i analítica de càrrega d'entrenament

Benvingut a la Natació Basada en Dades

SwimAnalytics transforma els teus entrenaments de natació en coneixements accionables utilitzant Critical Swim Speed (CSS), Training Stress Score (sTSS) i mètriques del Performance Management Chart (PMC). Aquesta guia et portarà des de la configuració inicial fins a l'analítica avançada de càrrega d'entrenament en 4 passos simples.

Inici Ràpid (5 Minuts)

1

Descarregar i Instal·lar

Descarrega SwimAnalytics des de l'App Store i concedeix permís per accedir a Apple Health. L'app sincronitza entrenaments de natació automàticament—no cal registre manual.

Descarregar App →
2

Realitzar Test CSS

Completa un cronometrat de 400m i 200m per establir el teu Critical Swim Speed. Aquesta és la base de totes les mètriques—sense CSS, sTSS i zones d'entrenament no es poden calcular.

Protocol Test CSS ↓
3

Introduir Resultats CSS

Introdueix els teus temps de 400m i 200m a l'app. SwimAnalytics calcula CSS, zones de ritme i personalitza totes les mètriques a la teva fisiologia. Actualitza cada 6-8 setmanes a mesura que millora la forma física.

4

Començar a Fer Seguiment d'Entrenaments

Neda amb Apple Watch i l'app Health. SwimAnalytics importa entrenaments automàticament, calcula sTSS, actualitza CTL/ATL/TSB i fa seguiment del progrés. No cal introduir dades manualment.

Protocol Complet de Test CSS

📋 Què Necessites

  • Accés a piscina: 25m o 50m (25yd acceptable)
  • Cronometratge: Cronòmetre, rellotge pace o Apple Watch
  • Temps d'escalfament: 15-20 minuts abans del test
  • Recuperació: 5-10 minuts entre proves
  • Esforç: Ritme màxim sostenible (no esprint total)

⏱️ Condicions Dia del Test

  • Descansat: Sense entrenament dur 24-48 hores abans
  • Hidratat: Ben hidratat, alimentació normal
  • Temp. piscina: 26-28°C ideal (evitar molt fred/calent)
  • Hora del dia: Quan normalment entrenes millor
  • Equipament: Igual que entrenament (ulleres, gorro, banyador)

Test CSS Pas a Pas

Escalfament

15-20 minuts

400-800m natació suau, exercicis tècnics i progressions. Inclou 2-3×50 a ritme creixent (60%, 75%, 85% esforç). Descansa 2-3 minuts abans del test.

Prova 1

400m Esforç Màxim

Sortida de paret (sense salt). Neda 400m al ritme més ràpid que puguis sostenir per tota la distància. Això NO és un esprint—dosifica't. Registra temps en format mm:ss (p.ex., 6:08).

Consell ritme: Apunta a parcials de 100m iguals. El segon 200m hauria de ser ≤ primer 200m (split negatiu ideal).
Recuperació

5-10 minuts

FASE CRÍTICA: Natació suau o descans complet. Espera fins que la freqüència cardíaca baixi de 120 ppm i respiració totalment recuperada. Recuperació insuficient = CSS inexacte.

Prova 2

200m Esforç Màxim

Sortida de paret (sense salt). Esforç màxim sostenible per 200m. Això hauria de sentir-se més dur per 100m que els 400m. Registra temps en format mm:ss (p.ex., 2:30).

Comprovació validació: Ritme 200m/100m hauria de ser 3-6 segons més ràpid que ritme 400m/100m. Si no, la recuperació va ser insuficient o el ritme incorrecte.
Refredament

10-15 minuts

300-500m natació suau, estiraments. Registra els teus temps immediatament—no confiïs en la memòria.

⚠️ Errors Comuns en Test CSS

  • Sortir massa ràpid en 400m: Resulta en explosió, CSS inexacte. Utilitza ritme constant.
  • Recuperació insuficient entre proves: La fatiga alenteix 200m, fent CSS artificialment ràpid → zones sobreentrenades.
  • Utilitzar sortides de salt: Afegeix avantatge 0.5-1.5s, esbiaixant càlculs. Sempre sortida de paret.
  • Testar quan fatigat: Càrrega d'entrenament pesada 24-48h abans = resultats deprimits. Testa descansat.
  • No registrar immediatament: La memòria no és fiable. Escriu temps abans del refredament.

Introduir Resultats CSS a SwimAnalytics

Pas 1: Obrir Configuració CSS

A l'app SwimAnalytics, ves a Configuració → Critical Swim Speed. Toca "Realitzar Test CSS" o "Actualitzar CSS".

Pas 2: Introduir Temps

Introdueix el teu temps de 400m (p.ex., 6:08) i temps de 200m (p.ex., 2:30). Utilitza el format exacte mostrat. Toca "Calcular".

Pas 3: Revisar Resultats

L'app mostra:

  • Velocitat CSS: 0.917 m/s
  • Ritme CSS: 1:49/100m
  • Zones d'entrenament: 7 zones personalitzades (Zona 1-7)
  • Línia base sTSS: Ara habilitat per tots els entrenaments

Pas 4: Desar i Sincronitzar

Toca "Desar CSS". L'app immediatament:

  • Recalcula zones d'entrenament
  • Actualitza retroactivament sTSS dels últims 90 dies
  • Ajusta càlculs CTL/ATL/TSB
  • Habilita analítica d'entrenament basada en zones

💡 Consell Pro: Test CSS Històric

Si ja coneixes el teu CSS de tests anteriors, pots introduir aquests temps directament. No obstant, per resultats més precisos, realitza un test nou cada 6-8 setmanes. El teu CSS hauria de millorar (ser més ràpid) a mesura que avança l'entrenament.

Comprendre les Teves Mètriques

Critical Swim Speed (CSS)

Què és: El teu ritme de llindar aeròbic—la velocitat més ràpida que pots sostenir durant ~30 minuts sense esgotament.

Què significa: CSS = 1:49/100m significa que pots mantenir ritme 1:49 per esforços de llindar sostinguts.

Com utilitzar: Base per totes les zones d'entrenament i càlcul sTSS. Actualitza cada 6-8 setmanes.

Aprendre CSS →

Zones d'Entrenament

Què són: 7 rangs d'intensitat basats en el teu CSS, des de recuperació (Zona 1) fins a esprint (Zona 7).

Què signifiquen: Cada zona apunta adaptacions fisiològiques específiques (base aeròbica, llindar, VO₂max).

Com utilitzar: Segueix prescripcions de zona per entrenament estructurat. L'app mostra temps-en-zona per cada entrenament.

Zones Entrenament →

Swimming Training Stress Score (sTSS)

Què és: Estrès d'entrenament quantificat combinant intensitat i durada. 1 hora a ritme CSS = 100 sTSS.

Què significa: sTSS 50 = recuperació fàcil, sTSS 100 = moderat, sTSS 200+ = sessió molt dura.

Com utilitzar: Fes seguiment d'sTSS diari/setmanal per gestionar càrrega d'entrenament. Apunta augment 5-10 sTSS per setmana màxim.

Guia sTSS →

CTL / ATL / TSB

Què són:

  • CTL: Chronic Training Load (forma física) - mitjana 42 dies sTSS
  • ATL: Acute Training Load (fatiga) - mitjana 7 dies sTSS
  • TSB: Training Stress Balance (forma) = CTL - ATL

Com utilitzar: TSB positiu = fresc/taper, TSB negatiu = fatigat. Competeix quan TSB = +5 a +25.

📊 Objectius Primera Setmana

Després d'introduir CSS i completar 3-5 entrenaments:

  • Comprova valors sTSS: Confirma que coincideixin amb percepció d'esforç (fàcil ~50, moderat ~100, dur ~150+)
  • Revisa distribució zones: Estàs passant 60-70% en Zona 2 (base aeròbica)?
  • Estableix CTL base: La mitjana sTSS de la primera setmana es converteix en línia base de forma física inicial
  • Identifica patrons: Quins entrenaments generen sTSS més alt? T'estàs recuperant adequadament?

Trajectòria Típica d'Usuari (Primeres 8 Setmanes)

Setmana 1-2: Establir Línia Base

  • Realitzar test CSS i introduir resultats
  • Completar 3-5 entrenaments normals
  • Observar valors sTSS i distribució zones
  • Establir CTL inicial (nivell forma física)
  • Objectiu: Comprendre mètriques, sense canvis encara

Setmana 3-4: Aplicar Zones

  • Utilitzar zones CSS en planificació entrenaments
  • Nedar intencionalment Zona 2 per sèries aeròbiques
  • Fer seguiment totals sTSS setmanals (apuntar consistència)
  • Monitoritzar TSB (hauria de ser lleugerament negatiu = entrenament)
  • Objectiu: Entrenar per zones, no per sensació

Setmana 5-6: Sobrecàrrega Progressiva

  • Augmentar sTSS setmanal 5-10% des de línia base
  • Afegir 1 sessió llindar (Zona 4) per setmana
  • CTL hauria de pujar gradualment (forma física millorant)
  • ATL pot augmentar en setmanes dures (normal)
  • Objectiu: Progressió controlada forma física

Setmana 7-8: Re-testar i Ajustar

  • Realitzar segon test CSS (hauria de ser més ràpid)
  • Actualitzar zones a l'app (ritme millora)
  • Comparar CTL Setmana 1 vs Setmana 8 (hauria de ser +10-20)
  • Revisar progrés: Estan baixant temps? Se sent més fàcil?
  • Objectiu: Validar efectivitat entrenament

✅ Indicadors d'Èxit

Després de 8 setmanes d'entrenament estructurat amb SwimAnalytics, hauries de veure:

  • Millora CSS: 1-3% ritme CSS més ràpid (p.ex., 1:49 → 1:47)
  • Augment CTL: +15-25 punts (p.ex., 30 → 50 CTL)
  • sTSS consistent: Totals setmanals dins 10-15% variància
  • Millor ritme: Parcials més iguals, millor calibratge esforç
  • Recuperació millorada: TSB cicla predictiblement (-10 a +5)

Resolució de Problemes i Preguntes Freqüents

El meu sTSS sembla massa alt/baix per l'esforç d'entrenament

Causa: CSS està desactualitzat o inexacte.

Solució: Re-testa CSS. Si vas testar quan fatigat o amb ritme pobre, CSS estarà malament. Un test CSS adequat és crític per totes les mètriques posteriors.

L'app mostra "CSS no configurat"

Causa: Test CSS no completat o no desat.

Solució: Ves a Configuració → Critical Swim Speed → Realitzar Test. Introdueix temps 400m i 200m, després toca Desar.

Entrenaments no sincronitzen des d'Apple Watch

Causa: Permisos app Health no concedits o entrenament no categoritzat com "Natació".

Solució: Comprova Configuració → Privacitat → Salut → SwimAnalytics → Permetre Lectura per Entrenaments. Assegura't que el tipus d'entrenament de l'Apple Watch estigui categoritzat com a "Natació Piscina" o "Natació Aigües Obertes".

CTL no augmenta malgrat entrenament consistent

Causa: Totals sTSS massa baixos o freqüència inconsistent.

Solució: CTL és mitjana ponderada exponencialment 42 dies. Puja lentament. Augmenta sTSS setmanal 5-10%, i mantingues 4+ entrenaments/setmana per creixement CTL consistent.

Amb quina freqüència hauria de re-testar CSS?

Recomanació: Cada 6-8 setmanes durant fases base/construcció. Re-testa després malaltia, lesió, pausa llarga, o quan zones se sentin consistentment massa fàcils/difícils.

Puc utilitzar SwimAnalytics per altres estils?

Sí, amb limitacions: CSS típicament es testa en crol. Per entrenaments estils/esquena/braça, sTSS es calcula basat en CSS crol. Considera realitzar tests CSS específics d'estil per més precisió.
