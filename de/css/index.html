Kritische Schwimmgeschwindigkeit (CSS)

Die Grundlage des datengesteuerten Schwimmtrainings

Was ist die Kritische Schwimmgeschwindigkeit (CSS)?

Die Kritische Schwimmgeschwindigkeit (CSS) ist die theoretische maximale Schwimmgeschwindigkeit, die Sie ohne Erschöpfung aufrechterhalten können. Sie entspricht Ihrem aeroben Schwellentempo, typischerweise entsprechend 4 mmol/L Blutlaktat und nachhaltig für etwa 30 Minuten. CSS wird mithilfe eines 400m- und 200m-Zeittests berechnet, um personalisierte Trainingszonen zu bestimmen.

Die Kritische Schwimmgeschwindigkeit (CSS) stellt die theoretische maximale Schwimmgeschwindigkeit dar, die Sie kontinuierlich ohne Erschöpfung aufrechterhalten können. Es ist Ihre aerobe Schwelle im Wasser – die Intensität, bei der Laktatproduktion und Laktatabbau im Gleichgewicht stehen.

🎯 Physiologische Bedeutung

CSS entspricht eng:

  • Laktatschwelle 2 (LT2) - Zweite ventilatorische Schwelle
  • Maximale Laktat-Steady-State (MLSS) - Höchster nachhaltiger Laktatspiegel
  • Funktionelles Schwellentempo (FTP) - Schwimm-Äquivalent zum Radfahren FTP
  • ~4 mmol/L Blutlaktat - Traditioneller OBLA-Marker

Warum CSS wichtig ist

CSS ist die grundlegende Kennzahl, die alle fortgeschrittenen Trainingsbelastungsanalysen ermöglicht:

  • Trainingszonen: Personalisiert Intensitätszonen basierend auf Ihrer Physiologie
  • sTSS-Berechnung: Ermöglicht eine genaue Quantifizierung des Trainingsbelastungswerts
  • CTL/ATL/TSB: Erforderlich für Performance Management Chart-Metriken
  • Fortschrittsverfolgung: Objektives Maß für Verbesserung der aeroben Fitness
⚠️ Kritische Abhängigkeit: Ohne einen gültigen CSS-Test können fortgeschrittene Trainingsbelastungsmetriken (sTSS, CTL, ATL, TSB) nicht berechnet werden. Ein ungenaues CSS wird alle nachfolgenden Trainingsanalysen verfälschen.

CSS-Testprotokoll

📋 Standardprotokoll

  1. Aufwärmen

    300-800m leichtes Schwimmen, Übungen und progressive Steigerungen zur Vorbereitung auf maximale Anstrengung.

  2. 400m Zeitfahren

    Maximale nachhaltige Anstrengung vom Abstoß (kein Sprung). Zeit auf die Sekunde genau aufzeichnen. Ziel: schnellste nachhaltige 400m.

  3. Vollständige Erholung

    5-10 Minuten leichtes Schwimmen oder vollständige Ruhe. Dies ist KRITISCH für genaue Ergebnisse.

  4. 200m Zeitfahren

    Maximale Anstrengung vom Abstoß. Zeit präzise aufzeichnen. Dies sollte schneller pro 100m sein als die 400m.

⚠️ Häufige Fehler

Unzureichende Erholung

Problem: Ermüdung verlangsamt die 200m-Zeit künstlich

Ergebnis: Berechnetes CSS wird schneller als die Realität, was zu übertrainierten Zonen führt

Lösung: Ruhen bis HF unter 120 bpm fällt oder bis die Atmung vollständig erholt ist

Schlechtes Tempo bei 400m

Problem: Zu schneller Start führt zu dramatischer Verlangsamung

Ergebnis: 400m-Zeit spiegelt nicht das wahre nachhaltige Tempo wider

Lösung: Gleichmäßige Splits oder negativer Split anstreben (zweite 200m ≤ erste 200m)

Verwendung von Sprungstarts

Problem: Fügt ~0,5-1,5 Sekunden Vorteil hinzu, verfälscht Berechnungen

Lösung: Immer Abstoß von der Wand verwenden

🔄 Wiederholungshäufigkeit

CSS alle 6-8 Wochen wiederholen, um Trainingszonen zu aktualisieren, wenn sich die Fitness verbessert. Ihre Zonen sollten progressiv schneller werden, wenn Sie sich an das Training anpassen.

CSS-Berechnungsformel

Formel

CSS (m/s) = (D₂ - D₁) / (T₂ - T₁)

Wobei:

  • D₁ = 200 Meter
  • D₂ = 400 Meter
  • T₁ = Zeit für 200m (in Sekunden)
  • T₂ = Zeit für 400m (in Sekunden)

Vereinfacht für Tempo pro 100m

CSS Tempo/100m (Sekunden) = (T₄₀₀ - T₂₀₀) / 2

Rechenbeispiel

Testergebnisse:

  • 400m Zeit: 6:08 (368 Sekunden)
  • 200m Zeit: 2:30 (150 Sekunden)

Schritt 1: CSS in m/s berechnen

CSS = (400 - 200) / (368 - 150)
CSS = 200 / 218
CSS = 0,917 m/s

Schritt 2: In Tempo pro 100m umrechnen

Tempo = 100 / 0,917
Tempo = 109 Sekunden
Tempo = 1:49 pro 100m

🎯 Vorteile zonenbasierten Trainings

Die Verwendung CSS-basierter Zonen verwandelt subjektives "Gefühls"-Training in objektives, wiederholbares Training. Jede Zone zielt auf spezifische physiologische Anpassungen ab:

  • Zone 2: Aufbau des aeroben Motors (60-70% des wöchentlichen Volumens)
  • Zone 3: Verbesserung der Renntempoeffizienz (15-20% des Volumens)
  • Zone 4: Laktatschwelle höher treiben (10-15% des Volumens)
  • Zone 5: Entwicklung von Spitzengeschwindigkeit und Kraft (5-10% des Volumens)

Wissenschaftliche Validierung

Wakayoshi et al. (1992-1993) - Grundlagenforschung

Kohji Wakayoshis bahnbrechende Studien an der Universität Osaka etablierten CSS als gültige, praktische Alternative zu Laktat-Labortests:

  • Starke Korrelation mit VO₂ an anaerober Schwelle (r = 0,818)
  • Exzellente Korrelation mit Geschwindigkeit bei OBLA (r = 0,949)
  • Sagt 400m-Leistung voraus (r = 0,864)
  • Entspricht 4 mmol/L Blutlaktat - maximale Laktat-Steady-State
  • Lineare Beziehung zwischen Distanz und Zeit (r² > 0,998)

Wichtige Veröffentlichungen:

  1. Wakayoshi K, et al. (1992). "Determination and validity of critical velocity as an index of swimming performance in the competitive swimmer." European Journal of Applied Physiology, 64(2), 153-157.
  2. Wakayoshi K, et al. (1992). "A simple method for determining critical speed as swimming fatigue threshold in competitive swimming." International Journal of Sports Medicine, 13(5), 367-371.
  3. Wakayoshi K, et al. (1993). "Does critical swimming velocity represent exercise intensity at maximal lactate steady state?" European Journal of Applied Physiology, 66(1), 90-95.

🔬 Warum CSS funktioniert

CSS stellt die Grenze zwischen den schweren und schweren Belastungsbereichen dar. Unterhalb von CSS bleiben Laktatproduktion und -abbau im Gleichgewicht—Sie können für längere Zeit schwimmen. Oberhalb von CSS sammelt sich Laktat progressiv an, bis Erschöpfung innerhalb von 20-40 Minuten eintritt.

Dies macht CSS zur perfekten Intensität für:

  • Festlegung nachhaltiger Renntempi für 800m-1500m-Wettkämpfe
  • Verschreibung von Schwellen-Intervalltraining
  • Überwachung von Verbesserungen der aeroben Fitness
  • Berechnung von Trainingsbelastung und Erholungsbedarf

Praktische Anwendungen

1️⃣ Trainingsbelastungsmetriken freischalten

CSS ist der Nenner in der Intensity Factor-Berechnung für sTSS. Ohne CSS können Sie Trainingsstress nicht quantifizieren oder Fitness-/Ermüdungstrends verfolgen.

2️⃣ Trainingszonen personalisieren

Generische Tempotabellen berücksichtigen nicht die individuelle Physiologie. CSS-basierte Zonen stellen sicher, dass jeder Schwimmer mit seiner optimalen Intensität trainiert.

3️⃣ Fitness-Progression überwachen

Alle 6-8 Wochen wiederholen. Verbessertes CSS (schnelleres Tempo) zeigt erfolgreiche aerobe Anpassung an. Stagnierendes CSS deutet darauf hin, dass das Training angepasst werden muss.

4️⃣ Rennleistung vorhersagen

CSS-Tempo approximiert Ihr nachhaltiges 30-Minuten-Renntempo. Verwenden Sie es, um realistische Ziele für 800m, 1500m und Freiwasser-Wettkämpfe zu setzen.

5️⃣ Schwellentrainings gestalten

Klassische CSS-Sets: 8×100 @ CSS-Tempo (15s Pause), 5×200 @ 101% CSS (20s Pause), 3×400 @ 103% CSS (30s Pause). Laktat-Clearance-Kapazität aufbauen.

6️⃣ Taper-Strategie optimieren

CSS vor und nach dem Taper verfolgen. Ein erfolgreicher Taper erhält oder verbessert CSS leicht, während Ermüdung reduziert wird (erhöhtes TSB).
