Comenzar con SwimAnalytics

Tu guía completa para el seguimiento del prestazione en nuoto, pruebas de CSS y análisis de carico de allenamento

Bienvenido a la Nuoto Basada en Datos

SwimAnalytics transforma tus allenamenti de nuoto en información práctica utilizando las métricas de Velocità Crítica de Nado (CSS), Puntuación de Estrés de Allenamento (sTSS) y Gráfico de Gestión del Prestazione (PMC). Esta guía te llevará desde la configuración inicial hasta el análisis avanzado de carico de allenamento en 4 pasos sencillos.

Inicio Rápido (5 Minutos)

1

Descargar e Instalar

Descarga SwimAnalytics desde el App Store y concede permiso para acceder a Apple Health. La app sincroniza los allenamenti de nuoto automáticamente: no se requiere registro manual.

Descargar App →
2

Realizar Test de CSS

Completa una test contrarreloj de 400m y 200m para establecer tu Velocità Crítica de Nado. Esta es la base de todas las métricas: sin CSS, no se pueden calcular sTSS ni zone de allenamento.

Protocolo de Test CSS ↓
3

Introducir Resultados de CSS

Introduce tus tiempos de 400m y 200m en la app. SwimAnalytics calcula CSS, zone de andatura y personaliza todas las métricas según tu fisiología. Actualiza cada 6-8 semanas a medida que mejora tu estado de forma.

4

Comenzar a Registrar Entrenamientos

Nada con Apple Watch y la app Health. SwimAnalytics importa automáticamente los allenamenti, calcula sTSS, actualiza CTL/ATL/TSB y realiza el seguimiento del progreso. No se necesita entrada manual de datos.

Protocolo Completo de Test de CSS

📋 Lo Que Necesitas

  • Acceso a piscina: Piscina de 25m o 50m (25yd aceptable)
  • Cronometraje: Cronómetro, reloj de pared o Apple Watch
  • Tiempo de calentamiento: 15-20 minutos antes de la test
  • Recupero: 5-10 minutos entre pruebas
  • Esfuerzo: Andatura máximo sostenible (no sprint total)

⏱️ Condiciones del Día de Test

  • Descansado: Sin allenamento duro 24-48 horas antes
  • Hidratado: Bien hidratado, alimentación normal
  • Temperatura piscina: 26-28°C ideal (evitar muy fría/caliente)
  • Hora del día: Cuando normalmente entrenas mejor
  • Equipamiento: Igual que en allenamenti (gafas, gorro, bañador)

Test de CSS Paso a Paso

Calentamiento

15-20 minutos

400-800m de nado suave, ejercicios técnicos y progresivos. Incluye 2-3×50 a andatura creciente (60%, 75%, 85% de esfuerzo). Descansa 2-3 minutos antes de la test.

Test 1

400m Esfuerzo Máximo

Salida desde pared (sin zambullida). Nada 400m al andatura más rápido que puedas sostener durante toda la distancia. Esto NO es un sprint: dosifica tu esfuerzo. Registra el tiempo en formato mm:ss (ej.: 6:08).

Consejo de andatura: Intenta mantener parciales de 100m uniformes. Los segundos 200m deberían ser ≤ primeros 200m (negativo ideal).
Recupero

5-10 minutos

FASE CRÍTICA: Nado suave o descanso completo. Espera hasta que la frecuencia cardíaca baje de 120 ppm y la respiración se recupere completamente. Recupero insuficiente = CSS inexacto.

Test 2

200m Esfuerzo Máximo

Salida desde pared (sin zambullida). Esfuerzo sostenible máximo durante 200m. Esto debería sentirse más duro por cada 100m que los 400m. Registra el tiempo en formato mm:ss (ej.: 2:30).

Verificación de validez: El andatura de 200m/100m debería ser 3-6 segundos más rápido que el andatura de 400m/100m. Si no es así, la recupero fue insuficiente o el andatura inadecuado.
Vuelta a la calma

10-15 minutos

300-500m de nado suave, estiramientos. Registra tus tiempos inmediatamente: no confíes en la memoria.

⚠️ Errores Comunes en la Test de CSS

  • Salir demasiado rápido en los 400m: Resulta en desfondamiento, CSS inexacto. Usa andatura uniforme.
  • Recupero insuficiente entre pruebas: La fatica ralentiza los 200m, haciendo que el CSS sea artificialmente rápido → zone de sobreentrenamiento.
  • Usar salidas con zambullida: Añade ventaja de 0.5-1.5s, sesgando cálculos. Siempre sal desde la pared.
  • Probar cuando estás fatigado: Carico de allenamento pesada 24-48h antes = resultados deprimidos. Test cuando estés fresco.
  • No registrar inmediatamente: La memoria no es fiable. Anota los tiempos antes de la vuelta a la calma.

Introducir Resultados de CSS en SwimAnalytics

Paso 1: Abrir Configuración de CSS

En la app SwimAnalytics, ve a Ajustes → Velocità Crítica de Nado. Toca "Realizar Test de CSS" o "Actualizar CSS".

Paso 2: Introducir Tiempos

Introduce tu tiempo de 400m (ej.: 6:08) y tiempo de 200m (ej.: 2:30). Usa el formato exacto mostrado. Toca "Calcular".

Paso 3: Revisar Resultados

La app muestra:

  • Velocità CSS: 0.917 m/s
  • Andatura CSS: 1:49/100m
  • Zone de allenamento: 7 zone personalizadas (Zona 1-7)
  • Línea base sTSS: Ahora habilitada para todos los allenamenti

Paso 4: Guardar y Sincronizar

Toca "Guardar CSS". La app inmediatamente:

  • Recalcula las zone de allenamento
  • Actualiza retroactivamente sTSS de los últimos 90 días
  • Ajusta los cálculos de CTL/ATL/TSB
  • Habilita análisis de allenamenti por zone

💡 Consejo Pro: Pruebas Históricas de CSS

Si ya conoces tu CSS de pruebas anteriores, puedes introducir esos tiempos directamente. Sin embargo, para obtener resultados más precisos, realiza una test nueva cada 6-8 semanas. Tu CSS debería mejorar (volverse más rápido) a medida que progresa el allenamento.

Comprender Tus Métricas

Velocità Crítica de Nado (CSS)

Qué es: Tu andatura de soglia aeróbico: la velocità más rápida que puedes sostener durante ~30 minutos sin agotamiento.

Qué significa: CSS = 1:49/100m significa que puedes mantener andatura de 1:49 en esfuerzos sostenidos de soglia.

Cómo usarlo: Base para todas las zone de allenamento y calcolo de sTSS. Actualiza cada 6-8 semanas.

Aprende CSS →

Zone de Allenamento

Qué son: 7 rangos de intensidad basados en tu CSS, desde recupero (Zona 1) hasta sprint (Zona 7).

Qué significan: Cada zona apunta a adaptaciones fisiológicas específicas (base aeróbica, soglia, VO₂máx).

Cómo usarlas: Sigue las prescripciones de zona para allenamento estructurado. La app muestra tiempo en zona para cada allenamento.

Zone de Allenamento →

Puntuación de Estrés de Allenamento en Nuoto (sTSS)

Qué es: Estrés cuantificado del allenamento combinando intensidad y duración. 1 hora a andatura CSS = 100 sTSS.

Qué significa: sTSS 50 = recupero fácil, sTSS 100 = moderado, sTSS 200+ = sesión muy dura.

Cómo usarlo: Rastrea sTSS diario/semanal para gestionar la carico de allenamento. Apunta a un aumento de 5-10 sTSS por semana como máximo.

Guía sTSS →

CTL / ATL / TSB

Qué son:

  • CTL: Carico de Allenamento Crónica (fitness) - promedio de 42 días de sTSS
  • ATL: Carico de Allenamento Aguda (fatica) - promedio de 7 días de sTSS
  • TSB: Balance de Estrés de Allenamento (forma) = CTL - ATL

Cómo usarlos: TSB positivo = fresco/en puesta a punto, TSB negativo = fatigado. Compite cuando TSB = +5 a +25.

📊 Objetivos de Tu Primera Semana

Después de introducir CSS y completar 3-5 allenamenti:

  • Verificar valores sTSS: Confirma que coincidan con la percepción de esfuerzo (fácil ~50, moderado ~100, duro ~150+)
  • Revisar distribución por zone: ¿Estás pasando 60-70% en Zona 2 (base aeróbica)?
  • Establecer CTL base: El promedio sTSS de tu primera semana se convierte en la línea base de fitness inicial
  • Identificar patrones: ¿Qué allenamenti generan mayor sTSS? ¿Te estás recuperando adecuadamente?

Recorrido Típico del Usuario (Primeras 8 Semanas)

Semana 1-2: Establecer Línea Base

  • Realizar test de CSS e introducir resultados
  • Completar 3-5 allenamenti normales
  • Observar valores sTSS y distribución por zone
  • Establecer CTL inicial (nivel de fitness)
  • Objetivo: Comprender métricas, sin cambios aún

Semana 3-4: Aplicar Zone

  • Usar zone CSS en planificación de allenamenti
  • Nadar intencionadamente en Zona 2 para series aeróbicas
  • Rastrear totales sTSS semanales (buscar consistencia)
  • Monitorear TSB (debería ser ligeramente negativo = entrenando)
  • Objetivo: Entrenar por zone, no por sensación

Semana 5-6: Sobrecarga Progresiva

  • Aumentar sTSS semanal un 5-10% desde la línea base
  • Añadir 1 sesión de soglia (Zona 4) por semana
  • CTL debería subir gradualmente (fitness mejorando)
  • ATL puede pico en semanas duras (normal)
  • Objetivo: Progresión controlada del fitness

Semana 7-8: Re-evaluar y Ajustar

  • Realizar segunda test de CSS (debería ser más rápida)
  • Actualizar zone en la app (el andatura mejora)
  • Comparar CTL Semana 1 vs Semana 8 (debería ser +10-20)
  • Revisar progreso: ¿Están bajando los tiempos? ¿Se siente más fácil?
  • Objetivo: Validar efectividad del allenamento

✅ Indicadores de Éxito

Después de 8 semanas de allenamento estructurado con SwimAnalytics, deberías ver:

  • Mejora en CSS: 1-3% más rápido en andatura CSS (ej.: 1:49 → 1:47)
  • Aumento de CTL: +15-25 puntos (ej.: 30 → 50 CTL)
  • sTSS consistente: Totales semanales dentro de 10-15% de varianza
  • Mejor dosificación: Parciales más uniformes, mejor calibración del esfuerzo
  • Recupero mejorada: TSB cicla predeciblemente (-10 a +5)

Solución de Problemas y Preguntas Frecuentes

Mi sTSS parece demasiado alto/bajo para el esfuerzo del allenamento

Causa: El CSS está desactualizado o es inexacto.

Solución: Vuelve a probar el CSS. Si probaste cuando estabas fatigado o dosificaste mal, el CSS será incorrecto. Una test de CSS adecuada es crítica para todas las métricas posteriores.

La app muestra "No hay CSS configurado"

Causa: Test de CSS no completada o no guardada.

Solución: Ve a Ajustes → Velocità Crítica de Nado → Realizar Test. Introduce ambos tiempos de 400m y 200m, luego toca Guardar.

Los allenamenti no se sincronizan desde Apple Watch

Causa: Permisos de la app Health no concedidos o allenamento no categorizado como "Nuoto".

Solución: Verifica Ajustes → Privacidad → Salud → SwimAnalytics → Permitir Lectura de Entrenamientos. Asegúrate de que el tipo de allenamento en Apple Watch sea "Nuoto en Piscina" o "Nuoto en Aguas Abiertas".

El CTL no aumenta a pesar del allenamento consistente

Causa: Totales sTSS demasiado bajos o frecuencia inconsistente.

Solución: CTL es un promedio ponderado exponencialmente de 42 días. Sube lentamente. Aumenta el sTSS semanal un 5-10%, y mantén 4+ allenamenti/semana para un crecimiento consistente del CTL.

¿Con qué frecuencia debería volver a probar CSS?

Recomendación: Cada 6-8 semanas durante las fases de base/construcción. Vuelve a probar después de enfermedad, lesión, pausa larga, o cuando las zone se sientan consistentemente demasiado fáciles/difíciles.

¿Puedo usar SwimAnalytics para otros estilos?

Sí, con limitaciones: El CSS típicamente se test en estilo libre. Para allenamenti de IM/espalda/braza, el sTSS se calcula basándose en el CSS de estilo libre. Considera realizar pruebas de CSS específicas por estilo para mayor precisión.
